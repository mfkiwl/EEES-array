s-sim - ES-SIMD simulator¶

SYNOPSIS¶

s-sim [options] filename

DESCRIPTION¶

The s-sim command simulates the cycle-level behaviour for a specified architecture.

The choice of architecture is determined by the -arch option or the -arch-cfg.

OPTIONS¶

The input files should be in Verilog memory initialization file (.imem_init and .dmem_init generated by s-as).

Other s-sim options are described below.

End-user Options¶

-help¶

Print a summary of command line options.

-arch=<arch>¶

Specify the architecture for which to generate assembly. If no architecture configuration file is specified by -arch-cfg, the default target architecture is “baseline”, which is a baseline 32-bit processor with 4 PE and explicit bypassing in both CP and PE.

If the architecture is different from the one specified by the configuration file, this option is ignored.

-arch-cfg=<arch.json>¶

Specify an architecture configuration file in JSON format.

-arch-param=<params>¶

Set specific architecture architecture parameters. A paramter is specified with a key:value pair. Multiple parameters are separated by comma.

For example, the following command sets the number of PE to 128 and the number of data memory entries to 2048:

s-sim -arch=baseline -arch-param pe:128,dmem-depth:2048
-dmem=<option:filename>¶

Specify data memory initialization file.

-dbin=<option:address:filename>¶

Specify binary file for data memory. The exact binary content will be copy to the address of the specified memory block.

-dump-dmem¶

Dump content of data memory.

-dump-dmem-prefix=<prefix>¶

Prefix of data memory dump files.

-imem=<filename>¶

Specify instruction memory initialization file.

-max-cycle=<string>¶

Maximum number of cycles to simulate.

-print-stat¶

Print simulation statistics.

-trace-level¶

Set simulation trace level (default=none).

none Disable trace generation.

brief Generate minimum level of trace.

pipeline Generate trace of pipeline events.

full Generate most detailed trace.

Debuging/Information Options¶

-quiet¶

Suppress any terminal output.

-verbose¶

Run in verbose mode.

-version¶

Display the version of this program.

EXIT STATUS¶

If s-sim succeeds, it will exit with 0. Otherwise, if an error occurs, it will exit with a non-zero value.
